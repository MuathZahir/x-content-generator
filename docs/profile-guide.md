# Profile Guide

Good penn AI output depends on specific profile context. Keep entries short, concrete, and easy for the model to apply.

## Context profile

Use this for stable background and opinions.

Example:

```txt
Who I am:
- Backend engineer
- Building AI developer tools
- Interested in agent workflows, specs, QA, and founder-led growth

Opinions:
- Agents need workflows, not just prompts
- Useful replies beat generic engagement
- Specs and verification matter more than clever phrasing
```

## Products and projects

Use separate blocks. Each block should include when to mention it and when not to.

Example:

```txt
SpecFlow
- Turns vague feature requests into specs, non-goals, and verification loops
- Mention when threads discuss AI coding agents, requirements, implementation drift, or QA
- Do not mention in general productivity or lifestyle threads

Agent Debrief
- Summarizes what coding agents changed inside a repo
- Mention when threads discuss reviewing AI-generated code, PR summaries, or developer workflow visibility
```

## Writing examples and tone

Describe the posture, not just adjectives.

Example:

```txt
Tone:
- direct
- practical
- specific
- willing to disagree politely
- no corporate enthusiasm

Style:
- short paragraphs
- one concrete example when useful
- avoid sounding like a launch announcement
```

## Forbidden phrases or behaviors

Use exact phrases and behaviors to filter.

Example:

```txt
Great point!
This is so true
Love this take
hashtags
links unless explicitly promoting
forced product plugs
```

## Never sound like this

Use full bad examples. These help the model avoid a broader style, not just exact words.

Example:

```txt
Great point! This is exactly why every founder needs to leverage AI to 10x their workflow.

Absolutely love this. We built a tool that solves this perfectly, check it out.
```

## Practical tips

- Add fewer, sharper product blocks instead of a long catalog.
- Include mention rules for each product.
- Put strong opinions in the profile; put promotion boundaries in products/projects.
- If replies sound generic, add better bad examples before adding more products.
- If product mentions feel forced, tighten the "mention when" lines.
